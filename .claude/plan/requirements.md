# 需求分析

> 关联文档：[技术调研](technical-research.md) · [架构设计](architecture.md) · [扩展性设计](extensibility.md)

---

## 1. 项目定位

**核心目标**：构建一个可真正落地使用的个人 AI 系统，而非技术 Demo。通过该项目实现从前端开发向 AI Agent 开发的转型。

**目标用户**：个人（或极小团队）日常使用。

**产品策略**：核心平台 + 垂直插件 — 先搭建完整的核心能力平台（多模型对话、Agent、Skills、MCP、RAG），各类垂直场景（代码助手、写作助手、翻译等）作为可插拔的插件快速接入。

---

## 2. 核心痛点分析

| 痛点 | 描述 | 解决方案 |
|------|------|---------|
| **多平台模型切换不便** | 不同任务需要不同模型，但每个平台都有独立的聊天界面 | 统一入口，一键切换模型/提供商 |
| **重复 Prompt 管理** | 每次新建对话都要重新写同样的 Prompt | **Skills 系统**：一次定义，随时调用 |
| **知识分散** | 个人文档、笔记、代码散落在各处，AI 无法感知 | **RAG 知识库**：文档上传即检索 |
| **工具能力受限** | 普通聊天无法执行代码、搜索网页、操作文件 | **Agent + MCP** 扩展工具链 |
| **垂直场景切换成本高** | 代码审查、写作辅助、翻译等场景需要不同平台 | **垂直场景插件**：一个平台覆盖所有 |

---

## 3. 功能分层

```
                    ┌──────────────────────┐
                    │   垂直场景插槽层      │  ← 代码助手、写作助手、翻译、客服...
                    │   (Slot Layer)       │
                    ├──────────────────────┤
                    │   扩展能力层          │
                    │   Skills │ MCP │ RAG │  ← 可扩展的工具和知识
                    ├──────────────────────┤
                    │   智能体层            │
                    │   Agent Runtime       │  ← ReAct 循环、工具调用
                    ├──────────────────────┤
                    │   核心基础层          │
                    │   多模型对话 + 流式    │  ← Phase 1 交付
                    └──────────────────────┘
```

---

## 4. 不做的事（明确边界）

- ❌ **不做本地模型部署**：仅调用外部 API
- ❌ **不做实时协作**：单人使用场景
- ❌ **不做复杂的工作流编排器**（如 Dify/Coze 级别）：Agent 能力聚焦在对话式交互
- ❌ **不做移动端 App**：初期仅 Web 端

> **注**：多租户不在初期范围内，但架构保留了扩展空间，详见 [扩展性设计](extensibility.md)。

---

## 5. Phase 1 功能优先级

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 多模型流式对话 | P0 | OpenAI、Anthropic、DeepSeek |
| 对话历史管理 | P0 | 新建/删除/切换对话 |
| 暗黑模式 | P0 | 跟随系统 / 手动切换 |
| Agent 基础能力 | P1 | ReAct 循环 + 内置工具 |
| Skills 系统 | P1 | 技能加载与注册 |
| MCP Client | P1 | HTTP/SSE 传输，接入外部工具 |
| RAG 知识库 | P2 | 文档上传、向量检索 |
| 垂直场景接入机制 | P2 | 场景模板 + 快速创建 |
