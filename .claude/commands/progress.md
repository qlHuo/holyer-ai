分析当前项目进度并更新规划。按以下步骤执行：

## 第一步：收集信息

1. 读取 `.claude/plan/roadmap.md` — 了解规划全貌和当前状态
2. 读取 `.claude/plan/status.md`（如果存在）— 了解上次分析结论
3. 读取 `.claude/plan/todo.md` — 了解推迟项列表
4. 运行 `git log --oneline --since="14 days ago"` — 了解近期变更
5. Glob `docs/decisions/*.md`，读取所有 ADR — 用于一致性检查
6. 读取 `docs/INDEX.md` — 了解最近新增的文档

## 第二步：分析现状

### 2a. 状态同步

对比 roadmap 中的状态标记与实际代码/文档现状，找出：
- 已完成但标记为 ⬜ 或 🔄 的任务 → 需要更新状态
- 当前 Phase 的完成百分比
- 下一步应该做的任务（按优先级排列，3-5 条）
- 是否有新增的阻塞或风险

### 2b. 规划纠偏（发现问题时才报告，无问题静默跳过）

**ADR 一致性检查**：对比每个 ADR 的决策结论与 roadmap 中的任务描述，检查矛盾。例如：
- ADR-008 否决了 Vercel AI SDK → roadmap 中不应有"集成 Vercel AI SDK"任务
- ADR-004 决定部署到 Cloudflare Pages → roadmap 中不应有"自建服务器部署"

发现矛盾时报告："⚠️ ADR-XXX 已决定 [X]，但 roadmap 中 [任务 Y] 与此矛盾，建议修正。"

**未规划工作检测**：对比近期 commit 和 dev-log 中的工作内容，检查是否存在于 roadmap 的某个任务中。如果有明显不在任何任务范围内的功能开发，报告：
"⚠️ 近期做了 [X]，但 roadmap 中没有对应任务。建议补充到 roadmap 或确认是否为临时性工作。"

**推迟项提醒**：检查 todo.md 中的条目，如果某条已存在超过 30 天且有相关 commit 涉及该领域，提示："💡 todo 中的 [X] 已推迟 30+ 天，近期有相关变更，是否考虑提升优先级？"

## 第三步：更新产出

### 3a. 如果 roadmap 状态过期，更新它

将已完成的任务状态改为 ✅，并在任务行末尾追加完成日期，如：
```
| 2.1 | Agent Runtime | ReAct 循环 + 上下文管理 | ✅ (2026-07-07) |
```

如果某个 Phase 全部完成，在 Phase 标题下追加完成日期。

如果发现 ADR 与 roadmap 矛盾，标记问题但不自动修改（需用户确认）。

### 3b. 覆盖写入 `.claude/plan/status.md`

```markdown
# 项目进度快照

> 更新于 YYYY-MM-DD

## 当前状态

**Phase X** — 完成度约 N%

## 近期完成

- [任务编号] 任务名（完成日期）
- ...

## 下一步

1. **[优先级] 任务名** — 一句话描述
2. ...
3. ...

## 阻塞 / 风险

（无则写"当前无阻塞项"）

## 推迟项

todo.md 中有 N 项待办，详见 [todo.md](todo.md)
```

约束：
- 全文不超过 30 行
- "近期完成"最多 5 条
- "下一步"最多 5 条，按优先级排序
- 不写技术细节——细节在 roadmap 里

## 第四步：汇报

用 5-8 行总结分析结果。如有纠偏发现，单独列出。最后提示：`status.md 已更新。`
