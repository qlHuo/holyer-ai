管理项目的低优先级待办事项，存储在 `.claude/plan/todo.md`。

## 执行逻辑

根据用户输入的参数决定操作：

### 无参数（默认）— 列出所有待办

读取 `.claude/plan/todo.md`，按分类展示所有待办项，带序号。例如：

```
## 🔧 功能优化
1. SSE 重连 — 网络闪断后自动恢复流式连接
2. 编辑重发 — 编辑已发送消息后重新发送
...

## 📝 工程改进
4. API 单元测试 — vitest + conversations CRUD 测试
```

然后提示："使用 `/todo add <描述>` 添加，`/todo done <序号>` 标记完成，`/todo promote <序号>` 提升到 roadmap。"

### `add <描述>`

1. 根据描述内容判断属于哪个分类（🔧 功能优化 / 📝 工程改进 / 💡 想法与实验）
2. 在 todo.md 对应分类末尾追加：`- [ ] **简短标题** — <描述>`
3. 汇报添加结果

### `done <序号>`

1. 先读取 todo.md 获取当前列表，按序号定位
2. 将该条目从 todo.md 中删除
3. 汇报："✅ 已完成并移除：[条目内容]"

### `promote <序号>`

1. 先读取 todo.md 获取当前列表，按序号定位
2. 将该条目从 todo.md 中删除
3. 提示："📋 已从 todo 移除：[条目内容]。请在 roadmap 中添加对应任务，然后运行 /progress 同步。"

## 分类说明

| 分类 | 适用场景 |
|------|---------|
| 🔧 功能优化 | 用户体验改进、新功能点子、交互优化 |
| 📝 工程改进 | 测试、CI/CD、代码质量、架构重构 |
| 💡 想法与实验 | 不确定要不要做、需要更多调研的点子 |
