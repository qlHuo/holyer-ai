### 关键经验

1. **Permissions 设计的核心原则**：allow 只放行**无破坏性**的操作，不给通配符过大权限。`Bash(git *)` 这种写法会匹配 `git push` 和 `git reset`，必须拆为具体安全子命令
2. **Hook 的跨平台兼容**：`jq` 不是 Windows 标配，用 `node -e` 内联 JSON 解析替代
3. **Chinese filenames trial**：plan/ 文档尝试过中文命名，最终回退为英文——CLI tab 补全和 Git 兼容性考虑
